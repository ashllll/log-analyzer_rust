# Milestone v1.0: Flutter 日志分析桌面应用 MVP

**Status:** ✅ SHIPPED 2026-03-01
**Phases:** 1-2
**Total Plans:** 7

## Overview

Flutter 全新实现的桌面日志分析应用 MVP 版本。通过 FFI 与现有 Rust 后端通信，实现了基础架构、工作区管理和文件导入功能。

## Phases

### Phase 1: 架构基础设施
**Goal**: Flutter 应用具备与 Rust 后端通信的基础设施，包括项目结构、共享服务、错误处理框架
**Depends on**: Nothing (first phase)
**Plans**: 4 plans

Plans:
- [x] 01-01-PLAN.md — FFI 桥接服务重构（纯 FFI 模式）
- [x] 01-02-PLAN.md — 错误处理框架（错误码 + ErrorView）
- [x] 01-03-PLAN.md — 启动流程与路由（Splash Screen + go_router）
- [x] 01-04-PLAN.md — Riverpod Provider 基础配置验证

**Details:**
- Flutter 项目结构搭建
- 移除 HTTP 客户端 (Dio)，改用纯 FFI 模式
- 错误处理框架：ErrorCodes 分类 + AppException + ErrorView 组件
- 启动流程：Splash Screen (含 FFI 初始化) + go_router 路由
- Riverpod Provider 状态管理配置

### Phase 2: 工作区与文件导入
**Goal**: 用户可以管理工作区、导入文件和压缩包
**Depends on**: Phase 1
**Plans**: 3 plans

Plans:
- [x] 02-01-PLAN.md — 工作区增强（键盘导航、最近优先排序）
- [x] 02-02-PLAN.md — 文件导入（拖放支持、导入进度显示）
- [x] 02-03-PLAN.md — 压缩包导入（ZIP/TAR/GZ/RAR/7Z 支持）

**Details:**
- 工作区管理：创建、打开、删除工作区
- 键盘导航支持（上下箭头 + 回车）
- 最近工作区优先排序（最近 3 个显示在最前）
- 文件夹拖放导入
- 压缩包导入（ZIP/TAR/GZ/RAR/7Z）
- 导入进度显示（ImportProgressDialog）
- 状态轮询机制（每 5 秒更新状态）

## Milestone Summary

**Key Decisions:**
- 使用 flutter_rust_bridge FFI 模式与 Rust 后端通信
- 使用 desktop_drop 实现跨平台拖放
- 使用 Riverpod 3.x 进行状态管理
- 工作区最近优先排序（SharedPreferences 持久化）

**Issues Resolved:**
- FFI 初始化时机问题：通过 Splash Screen 延迟初始化解决
- 用户友好错误显示：通过 ErrorView 组件统一处理

**Issues Deferred:**
- 后端 list_archive 命令未实现（压缩包预览为空）
- 后端选择性解压未支持

**Technical Debt Incurred:**
- 压缩包预览对话框显示空列表（需后端支持）
- 导入进度显示为模拟数据（需后端事件流）
