# 功能概览

Log Analyzer 是一个本地桌面应用，重点处理“数据量大、来源杂、不能随意上传”的日志分析场景。

## 核心能力

### 工作区

每次导入都会进入一个工作区上下文。工作区关联虚拟路径、内容哈希、元数据、索引状态与监听状态，让后续搜索不再依赖原始目录结构。

### 混合搜索引擎

查询规划器根据表达式选择 Memchr、Aho-Corasick、标准 Regex 或 Fancy look-around 路径。批处理执行和磁盘结果分页让大结果集保持可控。

### 组合过滤

- `|` 分隔多个 OR 关键词
- 时间范围
- `E / W / I / D` 日志级别
- 文件名与虚拟路径模式
- 标准正则与 look-around 查询

### 归档提取

支持 ZIP、TAR、GZ、7Z，以及启用对应 feature 时的 RAR。提取过程包含路径穿越与符号链接防护，并能继续处理嵌套归档。

### 内容寻址存储

日志内容按 SHA-256 存放在 CAS 中；SQLite 元数据记录工作区、文件、虚拟路径和索引信息。相同内容可在多个工作区之间复用。

### 实时监听

目录型工作区可以监听新增或变化的日志文件。`FileTailer` 与 `WatcherRunner` 管理增量读取和生命周期，前端通过 Tauri Events 接收状态更新。

### 关键词组

把常见告警词、组件名与故障模式整理为启用/禁用的关键词组，供搜索过滤和高亮重复使用。

## 明确边界

- 桌面端使用 Tauri Events 与 invoke，不提供 WebSocket 服务。
- 默认运行时不调用云端日志服务。
- 分布式 / 远程工作区目前不是主产品路径，相关评估见[分布式工作区评估](../architecture/DISTRIBUTED_WORKSPACE_ASSESSMENT.md)。
- 查询能力遵循当前 Rust 引擎的语法与 ReDoS 安全校验，不等同于任意 PCRE 实现。
