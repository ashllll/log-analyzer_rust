# v1.3 集成研究：自定义过滤器、多工作区标签页与日志级别统计

**项目**: log-analyzer_rust
**版本**: v1.3
**研究日期**: 2026-03-07
**总体置信度**: HIGH

## 执行摘要

本研究报告针对桌面日志分析应用 v1.3 版本的三个核心功能（自定义保存过滤器、多工作区标签页、日志级别统计）的集成模式进行深入分析。基于对现有 Riverpod 状态管理、FFI 桥接服务、FilterPalette 组件和路由系统的全面研究，提出了一套完整的技术集成方案。

**核心发现**：
- 现有 `SearchQueryProvider` 和 `WorkspaceStateProvider` 已建立成熟的 Riverpod 模式，可直接复用于新功能
- `BridgeService` 的单例模式和延迟初始化设计为新 FFI 接口提供了良好模板
- 过滤器、标签页和统计功能共享工作区上下文，需要统一的全局状态协调
- 推荐采用「功能模块化 + 全局状态协调」的混合架构

### 2.2 自定义过滤器状态管理

**推荐架构**：基于现有 `SearchQueryProvider` 扩展

```dart
// 复用 SearchTerm 模型扩展过滤器
part 'saved_filter.dart';

/// 保存的过滤器模型
class SavedFilter {
  final String id;
  final String name;
  final String? description;
  final String workspaceId;
  final List<SearchTerm> terms;
  final ffi.QueryOperatorData globalOperator;
  final TimeRange? timeRange;
  final List<String> levels;
  final String? filePattern;
  final bool isDefault;
  final int usageCount;
  final DateTime createdAt;
  final DateTime? lastUsedAt;
}

/// 过滤器 Provider
@riverpod
class SavedFilters extends _$SavedFilters {
  @override
  AsyncValue<List<SavedFilter>> build(String workspaceId) {
    Future.microtask(() => _loadFilters());
    return const AsyncLoading();
  }

  Future<void> _loadFilters() async {
    final bridge = ref.read(bridgeServiceForQueryProvider);
    final ffiFilters = await bridge.getSavedFilters(workspaceId);
    state = AsyncData(ffiFilters.map(_fromFfi).toList());
  }

  Future<void> saveFilter(SavedFilter filter) async {
    // 乐观更新
    final previous = state.value ?? [];
    state = AsyncData([...previous, filter]);

    // 后端同步
    final bridge = ref.read(bridgeServiceForQueryProvider);
    await bridge.saveFilter(filter.toFfi());
  }

  Future<void> deleteFilter(String filterId) async { ... }

  void applyFilter(String filterId) {
    // 将过滤器应用到 SearchQuery
    final filter = state.value?.firstWhere((f) => f.id == filterId);
    if (filter != null) {
      ref.read(searchQueryProvider.notifier).applyTerms(filter.terms);
      ref.read(searchQueryProvider.notifier).setGlobalOperator(filter.globalOperator);
    }
  }
}

/// 活动过滤器 Provider
@riverpod
class ActiveFilter extends _$ActiveFilter {
  @override
  SavedFilter? build(String workspaceId) {
    return null; // 无活动过滤器
  }

  void setActive(SavedFilter? filter) {
    state = filter;
    if (filter != null) {
      // 更新使用统计
      ref.read(savedFiltersProvider(workspaceId).notifier).updateUsage(filter.id);
    }
  }

  void clearActive() {
    state = null;
  }
}
```

### 2.3 日志级别统计状态管理

**推荐架构**：独立 Provider + 缓存策略

```dart
/// 日志级别统计模型
class LogLevelStats {
  final Map<String, int> counts;      // 级别 -> 数量
  final Map<String, double> percentages; // 级别 -> 百分比
  final int total;
  final DateTime? lastUpdated;
}

/// 日志级别统计 Provider
@riverpod
class LogLevelStats extends _$LogLevelStats {
  @override
  AsyncValue<LogLevelStats> build(String workspaceId) {
    // 首次加载
    Future.microtask(() => _loadStats());
    return const AsyncLoading();
  }

  Future<void> _loadStats() async {
    try {
      final bridge = ref.read(bridgeServiceForQueryProvider);
      final stats = await bridge.getLogLevelStats(workspaceId);
      state = AsyncData(stats);
    } catch (e, st) {
      state = AsyncError(e, st);
    }
  }

  /// 刷新统计
  Future<void> refresh() async {
    await _loadStats();
  }

  /// 获取指定级别的统计
  int getCount(String level) {
    return state.value?.counts[level] ?? 0;
  }
}

/// 统计刷新控制器
@riverpod
class StatsRefreshController extends _$StatsRefreshController {
  Timer? _refreshTimer;

  @override
  void build() {
    ref.onDispose(() {
      _refreshTimer?.cancel();
    });
  }

  /// 启动自动刷新
  void startAutoRefresh(Duration interval) {
    _refreshTimer?.cancel();
    _refreshTimer = Timer.periodic(interval, (_) {
      final workspaceId = ref.read(appStateProvider).activeWorkspaceId;
      if (workspaceId != null) {
        ref.read(logLevelStatsProvider(workspaceId).notifier).refresh();
      }
    });
  }

  void stopAutoRefresh() {
    _refreshTimer?.cancel();
  }
}
```

## 5. 性能优化策略

### 5.1 状态重建优化

**使用 `select()` 避免不必要重建**：

```dart
// 不推荐：整个 Provider 变化时重建
final filterCount = ref.watch(savedFiltersProvider(workspaceId));

// 推荐：只关心数量变化
final filterCount = ref.watch(
  savedFiltersProvider(workspaceId).select((filters) => filters.length),
);

// 推荐：只关心特定过滤器
final specificFilter = ref.watch(
  savedFiltersProvider(workspaceId).select(
    (filters) => filters.firstWhere((f) => f.id == filterId),
  ),
);
```

### 5.2 统计缓存策略

```dart
@riverpod
class LogLevelStats extends _$LogLevelStats {
  // 缓存配置
  static const _cacheDuration = Duration(seconds: 30);
  DateTime? _lastFetch;

  @override
  AsyncValue<LogLevelStats> build(String workspaceId) {
    Future.microtask(() => _loadStats());
    return const AsyncLoading();
  }

  Future<void> _loadStats() async {
    // 检查缓存
    if (_lastFetch != null &&
        DateTime.now().difference(_lastFetch!) < _cacheDuration &&
        state.hasValue) {
      return; // 使用缓存数据
    }

    // 重新加载
    final bridge = ref.read(bridgeServiceForQueryProvider);
    final stats = await bridge.getLogLevelStats(workspaceId);
    _lastFetch = DateTime.now();
    state = AsyncData(stats);
  }
}
```

## 6. 功能间集成点

### 6.1 过滤器与标签页

当用户在不同标签页切换时，过滤器状态需要保持：

```dart
// 标签页切换时保存/恢复过滤器状态
ref.listen(activeTabProvider, (previous, next) {
  if (previous != null) {
    // 保存当前标签页的过滤器状态
    final currentFilter = ref.read(activeFilterProvider);
    _saveTabFilterState(previous.id, currentFilter);
  }
  if (next != null) {
    // 恢复新标签页的过滤器状态
    final savedFilter = _loadTabFilterState(next.id);
    ref.read(activeFilterProvider.notifier).setActive(savedFilter);
  }
});
```

### 6.2 过滤器与搜索

过滤器应用时触发搜索更新：

```dart
// SavedFilters Provider 中
void applyFilter(String filterId) {
  final filter = state.value?.firstWhere((f) => f.id == filterId);
  if (filter != null) {
    // 1. 更新 SearchQuery
    ref.read(searchQueryProvider.notifier).clearKeywords();
    for (final term in filter.terms) {
      ref.read(searchQueryProvider.notifier).addKeyword(term.value);
    }

    // 2. 更新 FilterOptions
    ref.read(filterOptionsProvider.notifier).setFilterOptions(
      FilterOptions(
        timeRange: filter.timeRange ?? const TimeRange(start: null, end: null),
        levels: filter.levels,
        filePattern: filter.filePattern,
      ),
    );

    // 3. 触发搜索
    ref.read(searchResultsProvider.notifier).executeSearch();
  }
}
```
