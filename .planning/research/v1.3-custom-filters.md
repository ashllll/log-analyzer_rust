# 自定义保存过滤器研究

**项目**: log-analyzer_rust
**版本**: v1.3
**研究日期**: 2026-03-07
**总体置信度**: MEDIUM

## 执行摘要

本研究报告针对桌面日志分析应用中的自定义保存过滤器功能提供UI模式和技术实现方案。基于对现有搜索历史系统（SearchHistoryProvider）和过滤器面板（FilterPalette）的分析，结合Flutter桌面应用的最佳实践，提出了一套完整的实现方案。

**核心发现**：
- 现有 `SearchHistoryProvider` 已建立搜索历史持久化模式，可复用其架构
- `FilterPalette` 已实现时间范围、日志级别、文件模式三种过滤条件
- 推荐采用「侧边栏快捷访问 + 对话框创建/编辑」的UI模式
- 过滤器存储应基于工作区级别，实现与搜索历史的类似架构

## 1. 过滤器UI模式

### 1.1 推荐模式：侧边栏快捷访问 + 模态对话框

这种模式结合了快速访问和完整配置功能，是桌面应用中最常见的过滤器UI模式。

**组件结构**：
```
┌─────────────────────────────────────────────────────────────┐
│  搜索栏                              [保存当前过滤器] 按钮   │
├─────────────┬───────────────────────────────────────────────┤
│             │                                               │
│  过滤器面板  │           主内容区域                          │
│  ┌────────┐ │                                               │
│  │Errors  │ │   ┌─────────────────────────────────────┐   │
│  │  only  │ │   │ 过滤器创建/编辑对话框               │   │
│  ├────────┤ │   │                                       │   │
│  │Last 24h│ │   │ 过滤器名称: [________________]      │   │
│  ├────────┤ │   │                                       │   │
│  │Critical│ │   │ 时间范围: [选择日期范围...]          │   │
│  │+ Errors│ │   │ 日志级别: [✓]ERROR [ ]WARN [ ]INFO   │   │
│  └────────┘ │   │ 关键词:   [________________]          │   │
│             │   │ 正则表达式: [ ] 启用                 │   │
│  [+ 新建]   │   │                                       │   │
│             │   │ [取消]              [保存过滤器]     │   │
│             │   └─────────────────────────────────────┘   │
└─────────────┴───────────────────────────────────────────────┘
```

**优势**：
- 用户可在侧边栏快速选择已保存的过滤器
- 对话框提供完整的过滤器配置界面
- 符合用户对桌面应用的心理模型（类似邮件客户端的过滤器）
- 支持快捷键操作

### 1.2 备选模式：搜索栏集成

在搜索栏中添加过滤器选择下拉菜单。

**适用场景**：过滤器数量较少（<10个），且用户更倾向于在搜索时直接选择。

**UI结构**：
```
┌──────────────────────────────────────────────────────────────────┐
│ [搜索...] [▼过滤器] [保存▼]                              [搜索] │
│                                                                │
│ 过滤器: [Errors only ▼]  ┌──────────────────────────────┐       │
│                          │ Errors only                  │       │
│                          │ Last 24 hours               │       │
│                          │ Critical + Errors           │       │
│                          │ ──────────────               │       │
│                          │ + 新建过滤器                 │       │
│                          └──────────────────────────────┘       │
└──────────────────────────────────────────────────────────────────┘
```

### 1.3 推荐决策

| 场景 | 推荐模式 | 理由 |
|------|----------|------|
| 过滤器数量 > 5 | 侧边栏模式 | 避免下拉菜单过于冗长 |
| 过滤器数量 1-5 | 搜索栏集成 | 界面更简洁 |
| 复杂过滤器配置 | 侧边栏模式 | 对话框提供更大配置空间 |

**本项目推荐**：侧边栏快捷访问模式

**原因**：
1. 项目已有侧边栏组件（FileTreeSidebar、ArchiveTreeSidebar）
2. 用户可能需要创建多个过滤器（如"仅错误"、"最近24小时"、"关键错误+堆栈跟踪"）
3. FilterPalette已有展开/折叠UI，可无缝集成

## 2. 过滤器持久化与管理

### 2.1 数据模型设计

参考现有 `SearchHistoryItem` 模型设计：

```dart
/// 自定义过滤器模型
class SavedFilter {
  /// 过滤器唯一标识
  final String id;

  /// 过滤器名称（用户友好名称）
  final String name;

  /// 过滤器描述（可选）
  final String? description;

  /// 工作区ID
  final String workspaceId;

  /// 关键词列表（对应SearchTerm）
  final List<SearchTerm> terms;

  /// 全局操作符
  final QueryOperatorData globalOperator;

  /// 时间范围
  final TimeRange? timeRange;

  /// 日志级别列表
  final List<String> levels;

  /// 文件模式
  final String? filePattern;

  /// 创建时间
  final DateTime createdAt;

  /// 最后使用时间
  final DateTime? lastUsedAt;

  /// 使用次数
  final int usageCount;

  /// 是否为默认过滤器
  final bool isDefault;

  /// 排序权重（用于显示顺序）
  final int sortOrder;
}
```

### 2.2 持久化架构

采用与 `SearchHistoryProvider` 类似的架构：

```dart
/// 过滤器存储 Provider
@riverpod
class SavedFilters extends _$SavedFilters {
  @override
  AsyncValue<List<SavedFilter>> build(String workspaceId) {
    Future.microtask(() => _loadFilters());
    return const AsyncLoading();
  }

  Future<void> _loadFilters() async {
    // 从FFI加载过滤器
    final ffiFilters = ffi.getSavedFilters(workspaceId: workspaceId);
    state = AsyncData(ffiFilters.map(_fromFfi).toList());
  }

  Future<void> saveFilter(SavedFilter filter) async {
    // 乐观更新
    final previous = state.value ?? [];
    state = AsyncData([...previous, filter]);

    // 后端同步
    ffi.saveFilter(filter.toFfi());
  }

  Future<void> deleteFilter(String filterId) async {
    // 乐观更新
    state = AsyncData(
      state.value!.where((f) => f.id != filterId).toList()
    );

    // 后端同步
    ffi.deleteFilter(filterId: filterId, workspaceId: workspaceId);
  }

  Future<void> updateFilterUsage(String filterId) async {
    // 更新使用次数和最后使用时间
    // 类似 SearchHistory.addSearchHistory 的模式
  }
}
```

### 2.3 Rust后端支持

需要新增以下FFI接口：

| 接口 | 功能 | 备注 |
|------|------|------|
| `get_saved_filters` | 获取工作区所有保存的过滤器 | 返回 `Vec<SavedFilterData>` |
| `save_filter` | 保存或更新过滤器 | 包含完整的过滤器数据 |
| `delete_filter` | 删除指定过滤器 | 按ID删除 |
| `get_default_filters` | 获取默认过滤器 | 系统预设过滤器 |

### 2.4 存储表结构（SQLite）

```sql
CREATE TABLE saved_filters (
    id TEXT PRIMARY KEY,
    workspace_id TEXT NOT NULL,
    name TEXT NOT NULL,
    description TEXT,
    terms_json TEXT NOT NULL,      -- SearchTerm JSON序列化
    global_operator TEXT NOT NULL,
    time_range_start TEXT,
    time_range_end TEXT,
    levels_json TEXT,              -- ["ERROR", "FATAL"]
    file_pattern TEXT,
    is_default INTEGER DEFAULT 0,
    sort_order INTEGER DEFAULT 0,
    usage_count INTEGER DEFAULT 0,
    created_at TEXT NOT NULL,
    last_used_at TEXT,
    FOREIGN KEY (workspace_id) REFERENCES workspaces(id)
);

CREATE INDEX idx_saved_filters_workspace ON saved_filters(workspace_id);
CREATE INDEX idx_saved_filters_sort_order ON saved_filters(sort_order);
```

## 4. 快速过滤器访问模式

### 4.1 侧边栏过滤器面板

位置：搜索页面左侧，与FilterPalette配合使用

```dart
class SavedFiltersSidebar extends ConsumerWidget {
  @override
  Widget build(BuildContext context, WidgetRef ref) {
    final filters = ref.watch(savedFiltersProvider(workspaceId));
    final currentFilter = ref.watch(activeFilterProvider);

    return Column(
      children: [
        // 标题栏
        Padding(
          padding: EdgeInsets.all(12),
          child: Row(
            children: [
              Icon(Icons.filter_alt, size: 18),
              SizedBox(width: 8),
              Text('保存的过滤器'),
              Spacer(),
              IconButton(
                icon: Icon(Icons.add, size: 18),
                onPressed: () => _showCreateDialog(context),
                tooltip: '新建过滤器',
              ),
            ],
          ),
        ),

        // 过滤器列表
        Expanded(
          child: filters.when(
            data: (filterList) => ListView.builder(
              itemCount: filterList.length,
              itemBuilder: (context, index) {
                final filter = filterList[index];
                return _FilterListItem(
                  filter: filter,
                  isActive: currentFilter?.id == filter.id,
                  onTap: () => _applyFilter(context, ref, filter),
                  onEdit: () => _showEditDialog(context, filter),
                  onDelete: () => _confirmDelete(context, filter),
                );
              },
            ),
            loading: () => Center(child: CircularProgressIndicator()),
            error: (e, _) => Text('加载失败: $e'),
          ),
        ),
      ],
    );
  }
}
```

### 4.2 过滤器列表项设计

```dart
class _FilterListItem extends StatelessWidget {
  final SavedFilter filter;
  final bool isActive;
  final VoidCallback onTap;
  final VoidCallback onEdit;
  final VoidCallback onDelete;

  @override
  Widget build(BuildContext context) {
    return Container(
      margin: EdgeInsets.symmetric(horizontal: 8, vertical: 2),
      decoration: BoxDecoration(
        color: isActive ? AppColors.primary.withValues(alpha: 0.1) : null,
        borderRadius: BorderRadius.circular(6),
        border: isActive ? Border.all(color: AppColors.primary) : null,
      ),
      child: ListTile(
        dense: true,
        leading: Icon(
          filter.isDefault ? Icons.star : Icons.filter_alt_outlined,
          size: 18,
          color: isActive ? AppColors.primary : AppColors.textMuted,
        ),
        title: Text(
          filter.name,
          style: TextStyle(
            fontWeight: isActive ? FontWeight.bold : FontWeight.normal,
          ),
        ),
        subtitle: Text(
          _buildSubtitle(filter),
          style: TextStyle(fontSize: 11, color: AppColors.textMuted),
          maxLines: 1,
          overflow: TextOverflow.ellipsis,
        ),
        trailing: PopupMenuButton(
          itemBuilder: (context) => [
            PopupMenuItem(
              value: 'edit',
              child: Row(
                children: [
                  Icon(Icons.edit, size: 16),
                  SizedBox(width: 8),
                  Text('编辑'),
                ],
              ),
            ),
            PopupMenuItem(
              value: 'delete',
              child: Row(
                children: [
                  Icon(Icons.delete, size: 16, color: AppColors.error),
                  SizedBox(width: 8),
                  Text('删除', style: TextStyle(color: AppColors.error)),
                ],
              ),
            ),
          ],
          onSelected: (value) {
            if (value == 'edit') onEdit();
            if (value == 'delete') onDelete();
          },
        ),
        onTap: onTap,
      ),
    );
  }

  String _buildSubtitle(SavedFilter filter) {
    final parts = <String>[];
    if (filter.terms.isNotEmpty) {
      parts.add('关键词: ${filter.terms.length}个');
    }
    if (filter.levels.isNotEmpty) {
      parts.add('级别: ${filter.levels.join(", ")}');
    }
    if (filter.timeRange != null) {
      parts.add('时间范围');
    }
    return parts.isEmpty ? '无过滤条件' : parts.join(' | ');
  }
}
```

### 5.1 主流应用过滤器UI模式

| 应用 | 过滤器UI模式 | 特点 |
