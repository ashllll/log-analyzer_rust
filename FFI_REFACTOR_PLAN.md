### 问题分析
1. **维护成本高**: 同时维护 React 和 Flutter 两套前端
2. **架构复杂**: Tauri IPC + HTTP API + FFI 三种通信方式
3. **依赖冗余**: Tauri 框架带来额外的依赖和复杂性
4. **构建复杂**: 需要构建 Tauri 和 Flutter 两个版本

## 重构步骤

### Phase 1: 准备工作
- [ ] 创建重构分支

- [ ] 更新 CI/CD 配置

- [ ] 移除 axum 依赖

- [ ] 完善 Flutter 功能
- [ ] 添加 Flutter 测试

### Phase 6: 清理和验证

## 风险评估

| 风险 | 可能性 | 影响 | 缓解措施 |
|------|--------|------|----------|
| FFI 性能问题 | 中 | 高 | 提前进行性能测试 |
| Flutter 功能缺失 | 中 | 中 | 完整功能清单比对 |
| 构建系统破坏 | 低 | 高 | 保持 CI/CD 配置备份 |
| 数据兼容性 | 低 | 高 | 保持存储层不变 |

## 成功标准

1. 所有功能通过 FFI 调用
2. Flutter 前端功能完整
3. 无 Tauri 依赖
4. 无 HTTP API 服务器
5. 测试通过率 95%+
6. FFI 调用延迟 < 1ms
