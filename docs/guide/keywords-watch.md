# 关键词与持续监听

关键词组解决“团队总是在搜索同一批故障信号”的问题；目录监听解决“日志仍在持续写入”的问题。两者可以组合，但生命周期相互独立。

## 建立关键词组

![关键词组管理](../assets/readme/keyword-groups.png)

可以按问题域组织关键词，例如：

| 分组 | 关键词示例 |
| --- | --- |
| 可靠性 | `timeout`、`retry exhausted`、`circuit breaker` |
| 容量 | `queue full`、`backpressure`、`out of memory` |
| 网络 | `connection refused`、`dns`、`reset by peer` |

禁用分组会保留配置，但不再参与搜索页过滤和日志高亮。适合暂时关闭噪声较高的历史规则。

## 持续监听目录

1. 导入一个目录型工作区。
2. 等待它进入 `READY`。
3. 在工作区页开启监听。
4. 保持应用运行，让文件变化进入当前工作区。
5. 返回搜索页，重新执行保存的查询或关键词组合。

## 实际边界

- 监听依赖原始目录持续可访问。
- 新增内容进入存储和元数据后，搜索结果不会在旧 `search_id` 中自动重写。
- 高频写入时，批处理和防抖会减少 UI 更新压力；任务与工作区事件仍通过 Tauri Events 投影到前端状态。
- 删除工作区或停止监听前，先确认没有仍需观察的本地日志流。
