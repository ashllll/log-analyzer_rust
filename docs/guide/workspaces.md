# 工作区与导入

工作区是 Log Analyzer 的核心用户边界：它把一组可搜索日志、内容存储、虚拟路径和任务状态组织在一起。

![工作区总览](../assets/readme/workspaces-overview.png)

## 工作区状态

工作区状态字段（`status`）的取值为 `READY` / `PROCESSING` / `OFFLINE` / `ERROR` / `PARTIAL`。界面主要展示以下状态：

| 状态 | 含义 | 建议操作 |
| --- | --- | --- |
| `PROCESSING` | 正在读取、解压、存储或建立索引 | 等待任务完成，查看任务页进度 |
| `READY` | 可执行搜索 | 进入 Search Logs |
| `ERROR` 等失败状态 | 导入、校验或监听过程失败 | 查看错误提示与[故障排查](../operations/troubleshooting.md) |

`WATCHING` 不是状态值，而是工作区卡片上的独立标签：工作区开启目录监听（`watching` 开关）时显示，与状态正交——例如一个工作区可以同时是 `READY` 且正在监听。

## 文件与目录导入

导入目录适合长期增长的服务日志；导入单文件适合临时调查。无论入口如何，后端都会通过导入流水线完成任务创建、路径校验、内容写入、元数据更新与最终状态收敛。

::: warning 路径与权限
导入源必须能被当前桌面进程读取。移动或删除原始文件不会自动删除已进入 CAS 的内容，但目录监听会失去来源。
:::

## 归档导入

归档会先进入安全提取流程，再把符合规则的日志文件送入存储。提取器会拒绝逃逸目标目录的路径，并对符号链接保持防护。

推荐做法：

- 保留事故包原始文件，便于复核。
- 对来源不可信的归档，先在隔离环境中处理。
- 深层嵌套包会增加导入时间；关注任务页而不是重复点击导入。

## 刷新与监听

**Refresh workspace** 用于主动重新检查来源；监听适合持续写入目录。监听不是持续自动搜索：文件变化进入工作区后，仍由用户发起新的搜索会话。

## 内容去重

CAS 使用 SHA-256 作为内容身份。相同字节内容即使来自不同路径，也只需保存一份对象；虚拟路径和工作区关系仍分别记录在元数据中。
