## 📝 文档维护

### 文档分类原则
- **architecture/** - 长期有效的架构和设计文档
- **guides/** - 面向用户的使用指南
- **development/** - 开发环境和工具文档
- **reports/** - 项目报告（当前有效）
- **reports/archive/** - 历史报告（已完成/过期）

### 归档规则
当报告或状态文档不再活跃时，应移动到 `reports/archive/` 目录。
